# 📚 使用指南

## 🌐 Web界面使用指南

### 🏠 主页仪表板 (`/`)

**功能概览：**
- 系统状态监控
- 论文统计信息
- 快速操作入口
- 最近活动记录

**主要操作：**
1. **立即爬取**: 点击"立即爬取"按钮开始爬取新论文
2. **查看统计**: 查看系统中的论文总数和分类统计
3. **快速导航**: 通过卡片快速访问各个功能页面

### 📄 论文管理 (`/papers`)

**功能说明：**
- 查看所有爬取的论文列表
- 搜索和筛选论文
- 查看论文详细信息
- 导出论文数据

**使用方法：**
1. **浏览论文**: 滚动查看论文列表，每张卡片显示标题、作者、摘要
2. **搜索论文**: 在搜索框中输入关键词进行实时搜索
3. **查看详情**: 点击论文标题或"查看详情"按钮进入论文详情页
4. **论文操作**: 在详情页可以查看AI摘要、研究洞察、BibTeX引用等

**搜索技巧：**
- 支持标题、作者、摘要的全文搜索
- 使用英文关键词效果更好
- 可以搜索特定的arXiv ID

### 🔤 关键词管理 (`/keywords`)

**功能说明：**
- 添加新的搜索关键词
- 管理现有关键词
- 切换不同关键词的数据库
- 查看每个关键词的论文数量

**使用方法：**
1. **添加关键词**: 在输入框中输入关键词，点击添加
2. **管理关键词**: 查看现有关键词列表，可以编辑或删除
3. **切换数据库**: 点击关键词切换到对应的数据库视图
4. **查看统计**: 每个关键词显示对应的论文数量

**关键词建议：**
- 使用英文技术术语
- 避免过于宽泛或过于具体的关键词
- 可以使用领域相关的重要概念或技术名称

### 🔄 论文比较 (`/compare`)

**功能说明：**
- 选择多篇论文进行比较分析
- AI生成深度比较报告
- 支持结构化输出

**使用方法：**
1. **输入论文ID**: 在文本框中输入要比较的论文arXiv ID，每行一个
2. **使用示例**: 点击"添加示例ID"按钮使用实际存在的论文
3. **开始比较**: 点击"开始比较"按钮提交分析请求
4. **查看结果**: 等待AI分析完成，查看结构化的比较报告

**ID格式要求：**
- 使用完整的arXiv ID，如：2510.25767
- 每行输入一个ID
- 至少需要两篇论文进行比较

### 📈 研究洞察 (`/insights`)

**功能说明：**
- AI生成的热门研究主题分析
- 研究趋势识别
- 领域发展动态

**使用方法：**
1. **查看洞察**: 阅读AI生成的最新研究洞察报告
2. **热门主题**: 查看当前热门的研究主题和关键词
3. **趋势分析**: 了解研究领域的发展趋势
4. **时间筛选**: 可以选择不同时间范围的分析

### ⚙️ 系统配置 (`/settings`)

**功能说明：**
- 配置DeepSeek API密钥
- 系统参数设置
- 调度任务管理
- 系统状态监控

**使用方法：**
1. **API配置**: 设置DeepSeek API密钥和基础URL
2. **测试连接**: 点击"测试连接"验证API配置
3. **调度管理**: 启动/停止定时任务，设置执行时间
4. **参数设置**: 调整每天爬取的论文数量等参数
5. **状态监控**: 查看系统运行状态和统计数据

### 📤 数据导出 (`/export`)

**功能说明：**
- 导出论文数据到不同格式
- 批量处理和下载
- 自定义导出选项

**使用方法：**
1. **选择格式**: 选择导出格式（JSON、CSV、BibTeX等）
2. **设置筛选**: 可以按时间、关键词等条件筛选
3. **执行导出**: 点击导出按钮生成文件
4. **下载文件**: 下载生成的导出文件

## 💻 命令行界面使用指南

### 启动CLI

```bash
python run.py --mode cli
```

### 可用命令

启动CLI后，可以使用以下命令：

| 命令 | 功能 | 示例 |
|------|------|------|
| `status` | 查看系统状态 | `status` |
| `run` | 手动执行一次爬取 | `run` |
| `keywords` | 管理搜索关键词 | `keywords` |
| `search <关键词>` | 搜索论文 | `search machine learning` |
| `insights [天数]` | 获取研究洞察 | `insights 7` |
| `compare <ID1> <ID2>` | 比较论文 | `compare 2510.25767 2510.25748` |
| `trending [天数]` | 查看热门主题 | `trending 7` |
| `export [格式]` | 导出数据 | `export json` |
| `help` | 显示帮助信息 | `help` |
| `exit` | 退出CLI | `exit` |

### CLI使用示例

```bash
# 查看系统状态
> status

# 手动爬取论文
> run

# 搜索特定主题的论文
> search deep learning

# 查看最近7天的研究洞察
> insights 7

# 比较两篇论文
> compare 2510.25767 2510.25748

# 导出JSON格式数据
> export json
```

## ⏰ 定时调度使用指南

```bash
python run.py --mode scheduler
```

在Web界面的"系统配置"页面可以：
- 设置每日执行时间
- 配置爬取参数
- 启动/停止调度服务
- 查看执行历史

### 调度功能

- **自动爬取**: 每天在指定时间自动爬取新论文
- **智能分析**: 自动为爬取的论文生成AI摘要
- **增量更新**: 只处理新的论文，避免重复
- **错误处理**: 自动处理异常情况并记录日志

## 🔧 故障排除

### 常见问题

**Q: 立即爬取功能没反应**
A:
1. 检查API密钥是否正确配置
2. 在系统配置页面测试API连接
3. 查看日志文件了解具体错误

**Q: 搜索没有结果**
A:
1. 确保已经爬取过论文
2. 尝试使用不同的关键词
3. 检查关键词拼写

**Q: 论文比较失败**
A:
1. 确保论文ID格式正确
2. 检查论文是否存在于数据库中
3. 使用"添加示例ID"测试功能

**Q: 研究洞察无法生成**
A:
1. 确保有足够的论文数据
2. 检查API配置和网络连接
3. 尝试手动爬取更多论文

### 日志查看

- **Web界面**: 在系统配置页面查看状态
- **日志文件**: 查看 `data/logs/arxiv_scraper.log`
- **错误日志**: 查看 `data/logs/error_YYYYMM.log`

### 性能优化

- **定期清理**: 定期清理旧的日志文件

- **API限制**: 合理设置每天的爬取数量
- **缓存利用**: 利用智能缓存减少API调用
